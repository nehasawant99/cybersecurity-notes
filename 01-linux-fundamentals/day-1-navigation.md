Day 1 - Linux Navigation

Today I learned basic Linux terminal commands:

pwd - present working directory

ls - files in the directory or folder

cd - change directory

mkdir - create a new directory

touch - create a new file

cat - print content inside the file

cp - copy content from a file

mv - move or rename a file

rm - remove file

I also created files and folders inside Kali Linux and practised navigating the filesystem.

Day 2 - Users and Permissions

Today I learned User and permissions commands:

whoami - current log - in user
id - display the user id or group id
who - current log-in into system
ls -l - listed files with detailed information (permission, owner, size, date)
chmod - change file or directory permission
chown - change the owner or group of file or directory


## Permission Types

r = read
w = write
x = execute

## Numeric Values

4 = read
2 = write
1 = execute

KEY LEARNING FROM THIS MODULE:

Why is this important?
Bad permissions can lead to:

Unauthorized access
Privilege escalation
Sensitive data exposure

Many CTFs and real-world vulnerabilities involve files that are writable or executable by the wrong users.
